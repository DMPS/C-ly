ProviderConfig

Provider-specific configuration properties for an orca.Session. (This describes a data type. It is not accessible as a global object.) Most of these advanced settings are not needed in most situations and have sensible default values, thus the application does not need to set them. Typically the application will only need to set conferenceFactoryURI (if advanced conferencing feature is available) and crypto (to specify whether DTLS or SDES should be used.)

Type:
  • object
Properties:
Name Type Description
interfaceType string Value for the signalling interface. Set as 'SIP-WS' for SIP over Websocket, 'REST-WS' for REST over Websocket. Default is SIP-WS. REST-WS was supported from ORCA ECMS load 29.14.01.xx and it can only work with IMS 13.1/ISC 24.1/IBC-4 R4.0 as a proof-of-concept.
sendAuthOnInitReg boolean If true, we include Authorization header in initial sip REGISTER. The default is false meaning we would not send authorization in initial register.
sendAuthOnReregDereg boolean If true, we include Authorization header in sip re-register and de-register messages. The default is true meaning we would send authorization in re-register and de-register.
reUseCallidInReregDereg boolean If true, we will re-use the same callId used on initial register, when sending re-register or de-register. The default is true, meaning we will re-use the same callId.
stun string The STUN server to use for calls. Defaults to blank (none).
bundle boolean If false, the line "a=group:BUNDLE" is removed from the SDP if present. If true, no change is made. Defaults to true.
crypto string Default is 'sdes-srtp'. "dtls-srtp" is used for Firefox as dtls-srtp is the only supported cryption for Firefox.
conferenceFactoryURI string Conference factory public ID, needed for conference calling feature.
expires string | number Value for the Expires header in the initial SIP REGISTER. Default is '600'.
addCodecs boolean If set to true, then if VP8 codec is missing from incoming SDP, then all codecs are replaced with the VP8, red, and ulpfec codecs. If false, no change is made. Default is false.
dtmf string Specify the DTMF method to use. Set as 'sip' for SIP INFO, 'inband' for inband, or 'both' for both. Default is both.
dtmfDuration string | number Duration of a DTMF tone, in milliseconds. Default is 100.
dtmfGap string | number Gap between DTMF tones, in milliseconds. Default is 70.
audioBandwidth string | number The target bandwidth for audio in kilobits per second. Default is unconstrained.
videoBandwidth string | number The target bandwidth for video in kilobits per second. Default is unconstrained.
dataBandwidth string | number The target bandwidth for MSRP over DataChannel in kilobits per second. Default is unconstrained.
audioCodecs string Comma-separated list of allowed audio codecs. If a value is given, then any audio codecs not in the list will be removed from SDP. If no value or an empty string is given, then no change is made. Default is no change.
videoCodecs string Comma-separated list of allowed video codecs. If a value is given, then any video codecs not in the list will be removed from SDP. If no value or an empty string is given, then no change is made. Default is no change.
persistentPC boolean If true, create a new PeerConnection whenever the MediaStream is changed during a call, for example during a video upgrade or downgrade. If false, use the same PeerConnection for the duration of the call. Default is false.
breaker boolean This option is only for interoperation with the webrtc2sip open source project. True to enable the RTCWeb Breaker, false for no change. Default is false.
stripExtraSSRC boolean If true, remove extra and unnecessary 'synchronization source' attributes (a=ssrc:) from SDP (offers or answers) which Chrome may have added in scenarios involving video upgrade or downgrade. Default is true.
confWorkaroundChrome boolean If true, enable workaround for conference call issue found in certain environments. Default is false.
useFirstCandidate boolean If true, use the first ICE candidate immediately without waiting for any more candidates to be gathered. This is useful to prevent delay in calling if candidate gathering is slow, particularly in Firefox. If false, wait for the end of ICE candidates. Default is false.
removeIPV6Candidates boolean If true, remove any IPV6 candidates from SDP. If false, no change is made. Default is false.
autoAnswerTime string | number Used for service continuity feature. How long (in seconds) after registration the client will auto-answer new call requests. This applies only to the first registration done after the page loads (which is presumed to be an auto-register) and to internal re-registrations done as part of service recovery. Default is 0.
maxRecoveryAttempts string | number Used for service continuity feature. How many attempts are made to re-establish the websocket connection after a network outage. Default is 5.
networkRetryInterval string | number Used for service continuity feature. How long the client waits between attempts to re-establish the websocket connection. Default is 5.
sendRegisterOnRecovery boolean Used for testing. If true, send a register or re-invite once the websocket connection has been restored. Default is false.
registerResponseTime string | number How long to wait (in seconds) for a register response before failing the websocket connection. Default is 0.
registerRefreshTime string | number How often (in seconds) to do register refreshes. Default is 0.
msidHandling boolean Select the type of handling on incoming msid in SDP. We may select from among: 'Strip Incoming MSID (default)', 'Generate/Replace Incoming MSID' or 'None, no applicable handling'.
enableMDSPsupport boolean If true, then this device and contact is sharing its PUID with other devices. This device is either the main devie or a secondary MDSP (Multiple Devices Shared PUID) device. Default is false.
secondaryDeviceId boolean String to be included in SIP Contact Header to identify this device and contact as an MDSP (Multiple Devices Shared PUID) Secondary device. Only applicable if enableMDSPsupport is true. AT&T is expected to set this to, mobility="fixed", while, Verizon is expected to set this to, +g.gsma.rcs.ipcall. Default is, mobility="fixed".
